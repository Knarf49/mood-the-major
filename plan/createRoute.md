# Create Route — React Router v8 (Framework Mode)

## Overview

This project uses **React Router v8 Framework Mode** (detected from `react-router.config.ts`, `app/routes.ts`, and route modules under `app/routes/`).

**How route resolution works:**

```
request GET /login
  → app/routes.ts: route("login", "routes/login.tsx")
    → app/routes/login.tsx (route module)
      → type-safe via ./+types/login (auto-generated by react-router typegen)
```

## Steps to Add a Route

### 1. Create the route module

Create `app/routes/<name>.tsx`:

```tsx
import type { Route } from "./+types/<name>";

export function meta({}: Route.MetaArgs) {
  return [
    { title: "Page Title" },
    { name: "description", content: "Page description." },
  ];
}

export default function Component() {
  return <h1>Hello</h1>;
}
```

**Route module conventions:**
- Import types from `./+types/<name>` for full type safety
- Export `meta` for page metadata (optional)
- Export `loader` for server data loading (optional)
- Export a default component for the rendered page

### 2. Register in `app/routes.ts`

```ts
import { type RouteConfig, route } from "@react-router/dev/routes";

export default [
  route("url-path", "routes/<name>.tsx"),
] satisfies RouteConfig;
```

### 3. Run type generation

```bash
npm run typecheck   # runs react-router typegen && tsc
```

This auto-generates `./+types/` files and validates types.

## Route Config Helpers

| Helper | Signature | Purpose |
|--------|-----------|---------|
| `index(file)` | single file | Renders at parent URL (default child) |
| `route(path, file)` | path + file | Basic route |
| `route(path, file, [children])` | path + file + children array | Nested routes with layout |
| `layout(file, children)` | file + children | Layout wrapper without URL segment |
| `prefix(prefix, routes)` | string + array | Path prefix for a group of routes |
| `:param` | dynamic segment | `route("users/:id", ...)` → `{ id: string }` |

## Auth Routes Example (what was done)

### Before — single `AuthPage` component (unrouted)

```
app/components/page/LoginPage.tsx   — LoginForm + SignupForm + mode toggle (no route)
app/routes.ts                      — only index("/")
```

### After — two separate routes

```
app/routes/login.tsx   — imports LoginForm, wraps in Card, links to /signup
app/routes/signup.tsx  — imports SignupForm, wraps in Card, links to /login
app/routes.ts          — index + route("login", ...) + route("signup", ...)
```

**Refactoring done:** `LoginPage.tsx` exports `LoginForm`, `SignupForm`, `loginAction`, `signupAction`, `FormState`, `initialState` so route modules can reuse forms and logic without duplication.

### Resulting route config

```ts
// app/routes.ts
import { type RouteConfig, index, route } from "@react-router/dev/routes";

export default [
  index("routes/home.tsx"),
  route("login", "routes/login.tsx"),
  route("signup", "routes/signup.tsx"),
] satisfies RouteConfig;
```

URL mapping:
- `/` → home.tsx
- `/login` → login.tsx
- `/signup` → signup.tsx
